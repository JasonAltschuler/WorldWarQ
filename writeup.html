Texture Mapping


aircraft1.scn
Texture mapping was attempted by modifying the LoadMaterial call in particleview.cpp. Four textures were used to texture our terrain triangle mesh: snow, dirt, grass, and rock. For "cliffs" in the terrain (steep mesh faces), a rock texture was applied. Otherwise, snow, dirt, and grass were used based on the average vertex height of the mesh face. Texture materials were initialized in Draw().

We implemented 5 different views, toggleable by the keys: 1, 2, 3, 4, and 5.

  • '1': Bird's eye view: this is the basic fixed camera view. (Camera position needs adjusting per scene).

    • Procedural Modeling


    aircraft1.scn
    aircraft1.scn
    We used procedural modelling to generate our terrain. We implemented a version of the diamond-square algorithm. First we used a python script to generate a flat NxN triangle mesh, where N is validated to be a power of 2. From there random heights are generated for the four corners. These four corners are used to interpolate a height value for the center of the mesh, onto which a random value is added. Then the midpoint heights between the four corners are calculated by averaging the two closest corner heights. No random value is added to these midpoint heights. The algorithm then proceeds recursively through the four quadrants created by the corners, midpoints, and center until every vertex in the mesh has a height.

    Collision Detection


    aircraft1.scn
    aircraft1.scn
    Collisions are used in the following settings:

  • Bullet hits aircraft: aircraft's hitpoints decrease. If hitpoints == 0, explode and respawn. Flash the aircraft red if hit.
  • Aircraft hits aircraft: both aircrafts explode and rspawn.
  • Aircraft hits scene graph (ground): aircraft explodes and respawns.

    Note: the only node in the scene graph is the ground mesh. Everything else is maintained separately.
    Note: we don't implement bullet hits scene to speed the game up. This is an "acceptable hack" because once the bullet hits the scene, it goes through it and is no longer visible to the player. Also, the bullets have lifetimes of 10 seconds, so will be deleted soon.

    Implementation: see particle.cpp and R3Aircraft.cpp. The collision view is ~2/3 taken from Assignments 3 and 4 code.
